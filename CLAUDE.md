# CLAUDE.md — MamaCare

## Проект

Дневник беременности — SPA-прототип. Frontend-only, без бэкенда и персистентности.
Годится для демо; реальное использование потребует добавления бэкенда и auth.

## Стек

### Frontend (`mamacare/src/`)
- React 19.2.0 + Vite 8 (beta)
- JSX (не TypeScript)
- Без UI-библиотек — всё на inline CSS-in-JS
- Состояние: `useState` в одном компоненте (`MamaCare.jsx`), живёт только в памяти браузера

### Внешние интеграции
- **Yandex AI Studio** — чат-агент, вызывается из браузера через Vite-прокси (обход CORS)
- API-ключ в `.env`, попадает в клиентский bundle — **виден в DevTools** (известная уязвимость прототипа)

### Backend / БД
- Отсутствуют полностью
- При перезагрузке страницы весь стейт теряется

  MamaCare.jsx   # Всё приложение: 941 строка

## Функциональность

### Ядро
- **Дневник** — ежедневный чекин:
  - 5 уровней настроения (emoji)
  - 8 предустановленных симптомов: тошнота, усталость, головная боль, отёки, изжога, боль в спине, нарушение сна, тревога
  - Произвольные заметки: самочувствие, питание, активность
- **Трекер недели** — пользователь указывает текущую неделю (1–42) при онбординге; приложение показывает триместр и центрирует историю вокруг неё

### Дополнительно
- **AI-анализ недели** — среднее настроение за 7 дней, частота симптомов, рекомендации. Сейчас мок, реального вызова AI нет
- **AI-чат** — реальный чат с Yandex AI. Три категории: физическое здоровье, эмоции, организация. Агент знает имя пользователя и текущую неделю
- **Система алертов** — если за последние 7 дней есть хотя бы один критический симптом (головная боль, отёки, тревога, нарушение сна) — показывает экран-предупреждение с контактом врача и кнопкой звонка
- **Карточка врача** — имя, специализация, телефон, клиника. Опционально заполняется при онбординге

## Ключевые решения

- **Один файл** — вся логика в `MamaCare.jsx`, намеренно для простоты прототипа
- **Нет персистентности** — осознанный выбор для MVP/демо; следующий шаг — localStorage или бэкенд
- **Inline CSS** — без внешних библиотек, всё стилизовано вручную
- **Vite-прокси** — единственный способ вызвать Yandex AI без бэкенда (обход CORS)
- **API-ключ в bundle** — известная проблема; решается добавлением бэкенда-прокси

### При команде "сохрани сессию"

1. **Создать лог** в `sessions/YYYY-MM-DD-название.md` с frontmatter:
   ```yaml
   ---
   tags: [session]
   date: YYYY-MM-DD
   ---
   ```
   Включить: что сделано, какие файлы изменены, ключевые решения, открытые вопросы.

2. **Обновить** `00-home/текущие-приоритеты.md` — убрать выполненное, добавить новое.

3. **Зафиксировать решения** в `knowledge/decisions/` если было принято архитектурное решение.

4. **Зафиксировать баги** в `knowledge/debugging/` если был найден и/или исправлен баг.

5. **Обновить** любые устаревшие заметки в `atlas/` если изменилась архитектура.
